How It Works

1. Create a Receipt: Drop your files into the generator. It computes cryptographic hashes (SHA-256) and creates a receipt JSON file.

2. Distribute: Share the receipt alongside your software. Anyone can verify it independently.

3. Verify: Drop the receipt into the verifier. It checks the cryptographic hash chain and optional signatures.

üî¨ The Science Behind Thiele Receipts

Thiele receipts leverage fundamental principles from cryptography and formal verification to create provably correct, mathematically verifiable proofs of software construction.

Mathematical Foundation

1. Cryptographic Hash Functions

At the core, we use SHA-256, a cryptographic hash function with these properties:

  • Deterministic: Same input always produces same output: H(x) = H(x)
  • One-way: Given H(x), computationally infeasible to find x
  • Collision-resistant: Infeasible to find x ‚â† y where H(x) = H(y)
  • Avalanche effect: Small input change causes completely different output
SHA256("hello") = 2cf24dba5fb0a30e26e83b2ac5b9e29e1b161e5c1fa7425e73043362938b9824

2. Hash Chains for Integrity

Each step in the construction builds on previous steps, creating a Merkle chain:

Step 1: H‚ÇÅ = SHA256(file‚ÇÅ) Step 2: H‚ÇÇ = SHA256(file‚ÇÇ) Step 3: H‚ÇÉ = SHA256(H‚ÇÅ ‚à• H‚ÇÇ) // ‚à• means concatenation Global: H_global = SHA256(all_steps)

This creates an immutable chain: changing any single byte cascades through the entire structure, making tampering mathematically detectable.

3. Digital Signatures (Optional)

Receipts can be signed using Ed25519 (Edwards-curve Digital Signature Algorithm):

  • Public-key cryptography: Sign with private key, verify with public key
  • Non-repudiation: Only the signer could have created this signature
  • Fast & secure: 128-bit security level, faster than RSA

Signature = Sign(private_key, H_global) ensures authenticity and authorship

The Thiele Machine: Zero-Trust Computing

This receipt system is part of the broader Thiele Machine project‚Äîan exploration of zero-trust, formally verified computing from first principles.

üéØ Core Principles

  1. Replace trust with proof: Instead of "trust this binary," we provide mathematical proof of its construction
  2. Reproducible determinism: Same inputs + same process = identical outputs, always
  3. Client-side verification: You verify proofs yourself‚Äîno need to trust servers or third parties
  4. Formal verification: Properties proven mathematically, not just tested empirically
