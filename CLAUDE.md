# Thiele Machine — Claude Code Instructions

## Style Guide

The voice and style guide for this repository is at `.claude/STYLE_GUIDE.md`.
Read it before editing any file — Coq proofs, thesis chapters, scripts, READMEs, or comments.
It defines Devon's voice, the forbidden word list, the verification checklist, and the full .v file audit tracker.

## Key rules (summary)

- Every sentence must sound like Devon wrote it. First person. Direct. No lectures.
- Forbidden: leverage, utilize, facilitate, robust, comprehensive, ensuring, furthermore, consequently, delve, underscore, noteworthy, streamline, "it is worth noting", "we present", "we propose", passive voice hiding the actor, em-dashes doing colon work.
- Every claim needs either a proof reference or a falsification condition.
- "I" for Devon's solo work. "We" only in the mathematical sense (reader and I working through this together).
- Nothing is "obviously" / "clearly" / "trivially" correct.

## Project structure

- Coq proofs: `coq/` — 200 files, zero Admitted in the active kernel tree
- Python VM: `thielecpu/vm.py` (generated by `scripts/forge_vm.py` — never edit directly)
- OCaml extraction: `coq/Extraction.v` → `build/thiele_core.ml`
- Verilog RTL: `thielecpu/hardware/rtl/thiele_cpu_kami.v` (generated via Kami/Bluespec pipeline)
- Thesis: `thesis/` — short thesis, math spec, and full 13-chapter thesis

## Generated files (never edit directly)

- `build/thiele_core.ml` — edit `coq/Extraction.v`, rebuild with `make -C coq`
- `thielecpu/vm.py` — edit `scripts/forge_vm.py`, regenerate
- `thielecpu/hardware/rtl/thiele_cpu_kami.v` — edit `coq/kami_hw/ThieleCPUCore.v`, rebuild with `scripts/kami_extract.sh`
